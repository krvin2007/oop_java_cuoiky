Sơ Đồ Lớp (Class Diagram)

Sơ đồ lớp (Class Diagram) chi tiết của toàn bộ hệ thống quản lý gara ô tô,
được thiết kế bằng ngôn ngữ Mermaid.js.

Sơ đồ hiển thị đầy đủ các lớp thực thể (Entity), bộ điều khiển (DAO),
các thuộc tính, phương thức và các mối quan hệ (Kế thừa, Liên kết...).

Nguyên Lý OOP: Tính Đóng Gói (Encapsulation)

Che giấu thông tin

Mọi thuộc tính trong các lớp thực thể (như Nguoi, Xe, HangMuc) đều được đặt phạm vi private. Việc này ngăn chặn các tác động thay đổi trực tiếp từ bên ngoài làm sai lệch dữ liệu.

Bắt buộc giao tiếp thông qua các hàm **Getter/Setter** công khai.

Kiểm duyệt dữ liệu (Validation)

Dữ liệu đầu vào được kiểm duyệt chặt chẽ ngay khi Setter hoạt động hoặc khi thực hiện hàm nhập nhapInfo(). Ví dụ: Họ tên không được rỗng, số điện thoại phải khớp regex 10 chữ số của nhà mạng VN.

Nguoi.java (Trích nguồn thực tế) Java OOP 
public abstract class Nguoi {
    // Các thuộc tính được đóng gói private
    private int id;
    private String name;
    private String phone;
    private String address;

    // Getter cung cấp quyền đọc có kiểm soát
    public String getPhone() { return this.phone; }

    // Kiểm tra dữ liệu đầu vào bằng Regex trong hàm nhập
    public void nhapInfo(Scanner sc) {
        System.out.print("Nhap so dien thoai: ");
        this.phone = sc.nextLine().trim();
        while (!this.phone.matches("^(03|05|07|08|09)\\d{8}$")) {
            System.out.print("SĐT khong hop le! Moi nhap lai: ");
            this.phone = sc.nextLine().trim();
        }
    }
}

Nguyên Lý OOP: Tính Kế Thừa (Inheritance)

Phân cấp Kế thừa tối ưu
  • Kế thừa Con người: Lớp chi trừu tượng Nguoi chứa các trường chung như id, name, phone, address. Các lớp con ChuXe và NhanVien kế thừa qua từ khóa extends để tái sử dụng mã nguồn.
  • Mở rộng Nhân sự: NhanVien tiếp tục được kế thừa bởi các nhân sự chuyên biệt: KyThuatVien (Thợ máy), KeToan (Kế toán), ThuKho (Thủ kho) và QuanLy (Quản lý).
  • Kế thừa Hạng mục: Lớp HangMuc (cha) được kế thừa bởi LinhKien (bổ sung số lượng tồn kho) và DichVu.
LinhKien.java kế thừa HangMuc.java Java Kế thừa 
// Kế thừa lớp cha HangMuc thông qua từ khóa extends
public class LinhKien extends HangMuc {
    // Thuộc tính riêng biệt mở rộng của lớp con
    private int soLuongTon;
    private String location;

    // Constructor lớp con gọi Constructor lớp cha
    public LinhKien(String ma, String ten, double donGia, int soLuongTon, String location) {
        super(ma, ten, donGia); // Gọi constructor lớp cha
        this.soLuongTon = soLuongTon;
        this.location = location;
    }
}

Nguyên Lý OOP: Tính Đa Hình (Polymorphism)

Ghi đè phương thức (Method Overriding)

Hành vi tinhThanhTien(int soLuong) được khai báo ở lớp cha HangMuc nhưng có cách xử lý đa hình ở các lớp con:

  • Lớp LinhKien nhân với số lượng thực tế:
    Đơn giá * Số lượng.
  • Lớp DichVu trả về đơn giá cố định (trọn gói):
    Đơn giá.
Nạp chồng phương thức (Overloading)

Thể hiện qua việc nạp chồng các Constructor (Constructor không tham số phục vụ chuyển đổi CSDL, Constructor có tham số để khởi tạo trực tiếp).

Sự khác biệt đa hình trong triển khai code Java Đa Hình 
// 1. Triển khai tinhThanhTien trong LinhKien.java
@Override
public double tinhThanhTien(int soLuong) {
    return getDonGia() * soLuong; // Đơn giá nhân số lượng
}

// 2. Triển khai tinhThanhTien trong DichVu.java
@Override
public double tinhThanhTien(int soLuong) {
    return getDonGia(); // Trả về đơn giá cố định
}

Nguyên Lý OOP: Tính Trừu Tượng (Abstraction)

Lớp & Phương thức trừu tượng
  • Abstract Class: Lớp Nguoi và HangMuc được khai báo abstract. Các lớp này đóng vai trò định nghĩa cấu trúc khung mẫu và không thể khởi tạo đối tượng trực tiếp.
  • Abstract Method: Phương thức getRoleDescription() và tinhThanhTien(...) không có phần thân, ép buộc các lớp con phải tự triển khai chi tiết nghiệp vụ.
Thiết kế Interface đồng bộ

Dự án sử dụng Interface Generic IRepository<T> để định nghĩa trừu tượng các thao tác quản lý dữ liệu (CRUD). Tất cả lớp DAO đều phải triển khai Interface này.

IRepository.java (Interface Trừu Tượng) Java Interface 
package com.mycompany.quanlygara.controller;
import java.util.List;

// Interface định nghĩa các phương thức trừu tượng
public interface IRepository<T> {
    void themMoi(T item) throws Exception;
    void capNhat(T item) throws Exception;
    void xoa(Object id) throws Exception;
    List<T> layTatCa() throws Exception;
    T layTheoId(Object id) throws Exception;
}

Cơ Chế Lưu Trữ MySQL & Kết Nối JDBC

Quản lý Kết nối JDBC

Thông qua lớp tập trung KetNoiCSDL.java để cấp phát đối tượng Connection. Dự án tích hợp thư viện MySQL Connector J Driver.

An toàn dữ liệu & Transactions
  • Ngăn chặn SQL Injection: Sử dụng PreparedStatement truyền tham số dạng ? để an toàn.
  • Database Transaction: Áp dụng cơ chế tắt tự động lưu dữ liệu:
    conn.setAutoCommit(false);
    Chỉ khi trừ kho linh kiện thành công và thêm hóa đơn chi tiết thành công mới thực hiện lưu DB: conn.commit();
    Nếu có bất kỳ lỗi nào xảy ra, hệ thống tự động hoàn tác: conn.rollback();.
PhieuSuaChuaDAO.java (Database Transaction) Java JDBC Transaction 
try (Connection conn = KetNoiCSDL.getConnection()) {
    conn.setAutoCommit(false); // Bắt đầu Transaction
    try {
        // Lệnh SQL 1: Trừ tồn kho linh kiện
        PreparedStatement psUpdate = conn.prepareStatement(
            "UPDATE linh_kien SET so_luong_ton = so_luong_ton - ? WHERE ma = ?");
        psUpdate.executeUpdate();

        // Lệnh SQL 2: Thêm mới chi tiết phiếu sửa chữa
        PreparedStatement psInsert = conn.prepareStatement(
            "INSERT INTO repair_order_details (...) VALUES (...)");
        psInsert.executeUpdate();

        conn.commit(); // Thành công -> Commit
    } catch (Exception e) {
        conn.rollback(); // Lỗi -> Rollback toàn bộ
        throw e;
    }
}

Ràng Buộc Nghiệp Vụ Tự Động Thông Minh

Phân tích Biển Số Xe Việt Nam

Hệ thống tích hợp bộ quy tắc phân loại 5 loại biển số xe bằng Regular Expression (Regex) cực kỳ chi tiết trong Xe.java.

  • Biển dân sự: Trắng, Vàng, Xanh. Kiểm tra cấu trúc 2 chữ số đầu, chữ cái Seri và định dạng số đuôi.
  • Biển quân đội: Chữ cái ký hiệu đơn vị (ví dụ: AA, VT, HQ) kèm 4-5 chữ số.
Tự động truy xuất Tỉnh Thành & Đơn vị

Nếu biển số hợp lệ, hệ thống tự đối chiếu mã vùng để điền thông tin Nơi cấp / Đơn vị quản lý (Ví dụ: 29 -> Hà Nội, 59 -> TP.HCM, VT -> Tập đoàn Viettel) giúp tối ưu nhập liệu.

Xe.java (Xác thực Biển Số bằng Regex) Java Regular Expression 
// Regex phân tích cấu trúc biển số xe dân sự VN
String civilRegex = "^([0-9]{2})([A-Z]{1,2}[0-9]?)[- ]?([0-9]{3,5}|[0-9]{3}\\.[0-9]{2})$";

if (inputPlate.matches(civilRegex)) {
    String provCode = inputPlate.substring(0, 2);
    String provName = getProvinceName(provCode); // Lấy tên tỉnh
    if (provName != null) {
        isValid = true;
        this.licensePlate = inputPlate;
        System.out.println("Nơi cấp: " + provName);
    } else {
        System.out.println("Mã tỉnh không tồn tại!");
    }
}

Giải Thuật Báo Cáo & Thuật Toán Sắp Xếp Map

SQL Aggregation

Để tính toán các chỉ số phức tạp (Doanh thu, Top xe sửa chữa, Top thợ tích cực), hệ thống sử dụng kết hợp các lệnh SQL: SUM(), COUNT(), JOIN và GROUP BY.

Tận dụng LinkedHashMap trong Java

Truy vấn cơ sở dữ liệu đã sắp xếp sẵn dữ liệu giảm dần nhờ lệnh ORDER BY count DESC. Tại lớp BaoCaoController, dữ liệu trả về được đưa vào cấu trúc LinkedHashMap.

LinkedHashMap duy trì chính xác thứ tự chèn phần tử, giúp View in báo cáo ra màn hình đúng thứ tự mà không cần tốn hiệu năng sắp xếp lại trong bộ nhớ Java.

BaoCaoController.java (Thống kê và sắp xếp) Java Collections & SQL 
public Map<String, Integer> getMostRepairedVehicles(int limit) {
    // LinkedHashMap để giữ nguyên thứ tự sắp xếp trả về từ SQL
    Map<String, Integer> result = new LinkedHashMap<>();
    String sql = "SELECT ro.bien_so, COUNT(ro.ma_phieu) as qty " +
                 "FROM phieu_sua_chua ro " +
                 "GROUP BY ro.bien_so " +
                 "ORDER BY qty DESC LIMIT ?"; // SQL sắp xếp sẵn
    try (Connection conn = KetNoiCSDL.getConnection();
         PreparedStatement ps = conn.prepareStatement(sql)) {
        ps.setInt(1, limit);
        ResultSet rs = ps.executeQuery();
        while (rs.next()) {
            result.put(rs.getString("bien_so"), rs.getInt("qty"));
        }
    }
    return result;
}

Xử Lý Ngoại Lệ Tùy Chỉnh (Custom Exceptions)

Định nghĩa Custom Exception

Hệ thống xây dựng các lớp ngoại lệ chuyên biệt mở rộng từ lớp Exception cơ sở:

  • InvalidPlateNumberException: Khi biển số xe không hợp quy chuẩn Việt Nam.
  • PartOutOfStockException: Khi số lượng yêu cầu trong phiếu vượt quá tồn kho thực tế.
  • DuplicateMaException: Khi thêm mới dịch vụ/linh kiện bị trùng mã.
Kỹ thuật lùi biến lặp (i--) độc đáo

Tại View, khi nhập dữ liệu lặp (ví dụ nhập danh sách 5 linh kiện), nếu DAO ném ngoại lệ (nhập trùng mã), khối catch sẽ bắt lỗi, hiển thị cảnh báo và thực hiện **i--** để lùi vòng lặp lại, cho phép người dùng nhập lại mà không bị mất lượt hay lỗi app.

ConsoleView.java (Bắt lỗi và lùi biến lặp) Java Exception Handling 
for (int i = 0; i < count; i++) {
    System.out.println("Nhập thông tin thứ " + (i+1));
    DichVu dv = new DichVu();
    dv.nhapInfo(sc);
    try {
        dichVuDAO.themMoi(dv);
    } catch (Exception e) {
        System.out.println("Lỗi: " + e.getMessage());
        i--; // Lùi vòng lặp để nhập lại
    }
}
